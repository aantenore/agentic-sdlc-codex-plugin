# Change delivery evidence

## What was asked?
State the originating user or system request in plain language and cite the canonical request record.

## Scope and non-goals
List the delivered boundary, explicit exclusions, and any approved deviations.

## Inputs
List requirements, contracts, decisions, assumptions, upstream artifacts, and source paths used.

## What changed?
Group code and knowledge-base changes by intent. Include affected files without dumping raw diffs.

## Why was it decided?
Record the chosen approach, the evidence-backed rationale, alternatives rejected, and trade-offs. Never include private chain-of-thought.

## Outputs
List durable code, documentation, records, and user-visible behavior produced.

## Verification
Map each acceptance criterion to tests, browser checks, security checks, and evidence paths with outcomes.

## Generated explanation
Provide a concise non-technical explanation authored by Codex from the recorded inputs, outputs, decisions, and evidence. Label inference explicitly and link all source records.

## Lineage
Link the requirement, story, contract version and approval, capability boundary, relevant traces, commits, handoffs, and output registry entry.
